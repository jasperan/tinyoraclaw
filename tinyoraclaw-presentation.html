Why Oracle AI Database?

What You Get

  • In-database ONNX embeddings (no external API calls)
  • VECTOR_DISTANCE() with COSINE similarity
  • ACID transactions across all 7 tables
  • Enterprise-grade security, backup, HA
  • JSON, CLOB, VECTOR column types natively

vs. SQLite (TinyClaw)

  • SQLite WAL: single-writer, no concurrent writes
  • No native vector type or embedding generation
  • No connection pooling
  • No row-level locking (SELECT FOR UPDATE)
  • No built-in replication or HA

FETCH FIRST 5 ROWS ONLY;

Team Orchestration

1. User message arrives via channel
↓
2. Leader agent processes + generates response with @teammate mentions
↓
3. Mentioned teammates execute in parallel (pending counter tracks completion)
↓
4. Leader assembles final response from teammate outputs

Per-conversation locking prevents race conditions when multiple messages arrive for the same thread.

Vector Search

How It Works

  • ONNX model loaded directly into Oracle (ALL_MINILM_L12_V2)
  • VECTOR_EMBEDDING() generates embeddings at INSERT and query time
  • VECTOR_DISTANCE() computes COSINE similarity
  • No external embedding API. No network round-trip. No extra cost.

Performance

0
External API Calls
95%
Target Accuracy

Embedding generation and similarity search happen in the same transaction. The data never leaves the database.

-- Remember + Recall in pure SQL
INSERT INTO TINY_MEMORIES (id, content, embedding)
VALUES (:id, :content, VECTOR_EMBEDDING(ALL_MINILM_L12_V2 USING :content));

SELECT content, VECTOR_DISTANCE(embedding, VECTOR_EMBEDDING(
  ALL_MINILM_L12_V2 USING :query), COSINE) AS score
FROM TINY_MEMORIES ORDER BY score FETCH FIRST 5 ROWS ONLY;

Design Patterns

Async HTTP Bridge

TS calls Python over HTTP. Clean process boundary. Independent scaling.

SKIP LOCKED Queue

SELECT FOR UPDATE SKIP LOCKED. No message double-processing. Atomic claim.

Session CLOB

Full session state serialized to CLOB. Restore on reconnect. No data loss.

Audit Trail

Sequence-numbered responses. Every agent action tracked. Full replay possible.

SSE Real-Time

Server-Sent Events from Hono to dashboard. Live updates without polling.

File Exchange Tags

Structured tags for cross-process file references. Clean media handoff.
